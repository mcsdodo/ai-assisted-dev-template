# Update Changelog

Add an entry to `CHANGELOG.md` after completing work.

**Key principle:** Update IMMEDIATELY when completing work. Don't batch changes.

## Usage

This command invokes the changelog skill. See `.claude/skills/changelog-skill/SKILL.md` for writing guidelines and examples.

## Quick Steps

1. Identify what changed from user's perspective
2. Choose category (Added, Changed, Fixed, Removed)
3. Add concise entry under `[Unreleased]`
4. Commit with the code changes
